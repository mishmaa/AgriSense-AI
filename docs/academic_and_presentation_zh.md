# AgriSense AI 智慧农业平台：毕业设计与展示材料（简体中文）

## 1. 摘要

AgriSense AI 是一个面向现代农业场景的全栈智慧农业平台，系统融合物联网传感器实时监测、人工智能作物决策、智能灌溉、植物病害识别、产量预测、肥料推荐、天气风险分析、农资市场、作物日历、无人机与卫星监测概念界面等功能。项目采用 React、Vite、TailwindCSS、FastAPI、PostgreSQL、SQLAlchemy、JWT、WebSocket、Scikit-learn 等技术构建，具备较强的工程完整性、可扩展性和展示价值。该系统不仅满足高校毕业设计对功能完整性和技术深度的要求，也具备面向作品集、GitHub 开源展示和实习面试的商业化产品质感。

## 2. 引言

随着农业数字化、精准农业和智慧农场的发展，传统农业管理方式逐渐难以满足大规模、高效率、低资源浪费的现代农业需求。农业生产过程受到土壤湿度、温度、空气湿度、降雨、光照、风速、土壤酸碱度、肥力水平、病虫害和市场需求等多维因素影响。AgriSense AI 通过软件工程、人工智能与物联网技术的结合，为农户和农业管理者提供一个可视化、实时化、智能化的农业管理平台。

## 3. 问题陈述

传统农业管理主要存在以下问题：

- 农田环境数据采集不连续，无法实时掌握土壤和气候变化。
- 灌溉决策依赖经验，容易造成缺水或水资源浪费。
- 作物选择、肥料使用和病害判断缺乏数据支撑。
- 农业生产计划分散，缺少统一的作物日历和任务提醒。
- 农场管理界面通常偏向基础 CRUD，缺乏真实产品级体验。

本项目旨在构建一个集数据采集、智能分析、实时预警和可视化决策于一体的智慧农业平台。

## 4. 项目目标

- 构建一个完整的前后端分离智慧农业平台。
- 实现 IoT 传感器实时数据模拟、存储和 WebSocket 推送。
- 实现作物推荐、灌溉预测、肥料推荐、病害检测、产量预测和天气建议等 AI 功能。
- 实现 JWT 登录认证和角色权限控制。
- 构建具有商业级视觉效果的响应式前端界面。
- 支持中文、英文和乌尔都语多语言切换，其中中文为默认语言。
- 提供 Docker 与 Docker Compose 部署方案。
- 输出适合高校答辩、GitHub 展示和个人作品集使用的完整文档。

## 6. 文献综述

智慧农业通常由感知层、网络层、平台层和应用层组成。感知层通过传感器采集土壤、水分、气候和作物状态数据；网络层通过 HTTP、MQTT、WebSocket 等协议实现数据传输；平台层负责数据存储、清洗和智能分析；应用层则通过可视化系统辅助农户做出决策。近年来，机器学习在作物推荐、产量预测、病害识别和灌溉优化中得到广泛应用。随机森林、梯度提升树、逻辑回归、卷积神经网络等算法在农业场景中具有较好的解释性和实用性。本项目结合这些思路，构建一个面向教学与展示的完整智慧农业系统原型。

## 8. 系统架构

系统采用前后端分离架构。前端负责交互界面、图表、动画和多语言展示；后端负责 API、认证、权限、数据库访问、AI 调用和 WebSocket 推送；数据库负责持久化用户、农场、传感器、读数、通知和 AI 结果；ML 模块负责模型训练和预测；IoT 模拟器负责生成实时农业传感器数据。

## 9. 数据库设计

核心数据表包括：

- users：用户信息、角色、语言偏好、联系方式。
- farms：农场基本信息、地理位置、面积、土壤类型。
- farm_zones：农场地块、作物、灌溉模式。
- sensors：传感器设备信息。
- sensor_readings：实时与历史传感器读数。
- irrigation_events：灌溉事件与用水记录。
- ai_recommendations：AI 推荐输入、输出与置信度。
- disease_detections：病害检测结果。
- notifications：系统通知、短信和邮件预警记录。
- marketplace_items：农资市场商品。
- crop_calendar_events：作物日历任务。

传感器读数表增加了农场、地块、传感器和时间的复合索引，以提升历史分析和仪表盘查询性能。

## 10. 前端说明

前端采用 React + TailwindCSS 构建，整体风格参考高端科技仪表盘，使用深绿色、玻璃拟态、粒子背景、流畅动画和响应式布局。系统支持移动端、平板和桌面端。通过 React Router 实现页面路由，通过懒加载优化首屏加载速度，通过 ErrorBoundary 提升异常容错能力，通过 Skip Link 和 reduced-motion 支持提升可访问性。

## 11. 后端说明

后端采用 FastAPI 构建 REST API 和 WebSocket 服务。系统使用 JWT 认证和基于角色的权限控制，使用 SQLAlchemy ORM 访问 PostgreSQL 数据库。后端加入 GZip 压缩、安全响应头、可信主机校验、CORS 配置、统一错误处理、结构化日志和短期缓存机制，提升生产环境质量。

## 12. AI 模块说明

AI 模块包括：

- 作物推荐：随机森林分类器，根据 NPK、温度、湿度、pH、降雨和土壤类型推荐作物。
- 灌溉预测：梯度提升分类器，根据土壤湿度、天气、水箱和作物阶段判断灌溉动作。
- 产量预测：随机森林回归器，根据作物、面积、天气、灌溉和土壤健康预测产量。
- 肥料推荐：随机森林分类器，根据营养元素缺失和作物阶段生成肥料计划。
- 天气建议：TF-IDF 与逻辑回归模型生成天气相关农业建议。
- 病害检测：基于叶片特征的分类模型，作为未来 CNN 图像模型的轻量化原型。
- 聊天助手：基于意图识别的农业问答助手，支持中文、英文和乌尔都语关键词。

## 13. IoT 模块说明

IoT 模拟器支持土壤湿度、温度、湿度、pH、水箱、降雨、光照和风速数据模拟。模拟逻辑考虑昼夜温度曲线、降雨事件、灌溉脉冲、光照变化和风速波动。数据可通过 HTTP 写入 FastAPI，也可通过 MQTT 适配器扩展到真实消息代理。后端接收数据后存储到 PostgreSQL，并通过 WebSocket 推送到前端。

## 20. 答辩问题与参考答案

问题：为什么选择 FastAPI？

回答：FastAPI 具有高性能、自动生成 OpenAPI 文档、原生支持异步和 Pydantic 数据校验等优势，非常适合构建实时数据和 AI 服务接口。

问题：系统如何实现实时更新？

回答：IoT 模拟器将传感器数据写入后端，后端存储数据后通过 WebSocket 向前端推送，前端无需轮询即可更新图表和传感器卡片。

问题：AI 模型如何集成到后端？

回答：模型通过 Scikit-learn Pipeline 训练，并使用 Joblib 序列化。后端 AI Engine 延迟加载模型，对 API 输入进行预测，并返回结果、置信度和解释。

问题：项目与普通 CRUD 系统相比有什么优势？

回答：本项目包含实时 IoT、WebSocket、AI/ML、数据可视化、多语言、语音助手、部署配置和高级 UI，具备更完整的工程体系和产品化体验。

问题：如何保证安全性？

回答：系统使用 JWT、密码哈希、RBAC、CORS 限制、安全响应头、环境变量密钥管理和可信主机校验等措施。
